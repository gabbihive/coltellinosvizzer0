Server Express

A web server with a built-in admin control panel, user management, and database browser — all in one place.

Node.js + Express PostgreSQL Prisma ORM Hosted on Render

? What is this?

This is a web application server — a program that runs 24/7 on the internet, waiting for visitors and responding to their requests.

Think of it like a restaurant: the server is the kitchen. When you visit a webpage (place an order), the server prepares the data you need (cooks your food) and sends it back to your browser (serves your plate).

This particular server comes with a control panel where the admin can manage users, browse the database, view logs, and change settings — all from a web browser.

⚙ How does it work?

Here's the journey of a request, step by step:

Your Browser → Render (cloud host) → Express (web server) → Prisma (ORM) → PostgreSQL (database)
  • 1. You visit a URL Your browser sends a request over the internet to this server's address.
  • 2. Express receives it Express is the framework that listens for requests and decides what to do with them — like a receptionist routing your call.
  • 3. Prisma talks to the DB If data is needed (like a list of users), Prisma translates the request into a database query. It's a translator between JavaScript and SQL.
  • 4. PostgreSQL stores data The actual database — a structured filing cabinet that holds all users, settings, and application data permanently.
  • 5. Response comes back The data flows back through the chain and arrives in your browser, usually in under a second.

★ The tech stack explained

  • Node.js The engine that lets JavaScript run outside a browser — it powers the server.
  • Express A lightweight framework that makes it easy to define routes (URLs) and handle requests.
  • Prisma An ORM (Object-Relational Mapper) — lets you work with the database using JavaScript instead of writing raw SQL.
  • PostgreSQL A powerful, open-source database. Think of it as a very organized spreadsheet that can handle millions of rows.
  • express-session Keeps you logged in. When you sign in, the server gives your browser a cookie to remember you.
  • Render The cloud platform where this server lives. It keeps the server running and gives it a public URL.

⚖ What can the admin do?

Behind the login screen, there's a control panel with four tabs:

  • Dashboard See server status (uptime, memory usage, Node version) and a live request log showing every API call.
  • Users Create, edit, and delete users. Each user has an email, optional name, and timestamps.
  • Database Browse any table in the database, view column types, check migration history — no SQL knowledge needed.
  • Settings Change the admin password, view environment variables, and check system information.

⚡ Dead Drop — anonymous paste bin

A zero-knowledge encrypted paste bin for sharing sensitive text. Your content is encrypted in the browser before it ever reaches the server — the server literally cannot read it.

  • Client-side encryption AES-256-GCM encryption happens entirely in your browser. The key lives in the URL fragment, which is never sent to the server.
  • Burn after reading Optionally destroy the message after it's read once. Gone forever.
  • No traces No logs, no IP tracking, no cookies, no metadata. Drops auto-expire.

⚠ How is it secured?

  • Password hashing Passwords are hashed with scrypt (a one-way function). Even if someone got the database, they can't reverse the password.
  • Timing-safe comparison Password checks use constant-time comparison to prevent timing attacks (where attackers measure response time to guess characters).
  • Secure cookies Session cookies are HTTP-only (JavaScript can't read them), SameSite (prevents cross-site attacks), and Secure (HTTPS only in production).
  • SQL injection protection Database table names are validated against the actual database before use. Prisma parameterizes all queries automatically.
