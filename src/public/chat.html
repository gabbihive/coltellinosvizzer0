How it works
1. A random room ID and 256-bit AES-GCM key are generated in your browser
2. The room ID is in the URL path — the server sees this (it's the room address)
3. The encryption key is in the URL fragment (#) — browsers never transmit this
4. Messages are encrypted client-side before being sent over WebSocket
5. The server relays opaque ciphertext between participants — it cannot decrypt anything
6. No messages are stored on the server — not in memory, not in a database
7. No IP addresses, cookies, user agents, or metadata are logged for chat routes
8. Rooms are destroyed the moment the last participant disconnects
9. Even if the server is compromised, past messages cannot be recovered — they never existed on disk
